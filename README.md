# NutriAI Web

NutriAI Web е версия на приложението, която работи директно в браузър и може да се качи в GitHub без Xcode, Mac или iOS компилация.

Дизайнът е mobile-first и е направен да се усеща като телефонно приложение: долна навигация, калориен пръстен, карти за макронутриенти, хидратация, камера, прогрес и чат.

Първото отваряне показва само профил за хранителен план. След запазване профилът се скрива и се отключват всички основни екрани.

Данните се пазят автоматично в браузъра на всеки потребител чрез `localStorage`: профил, хранения, вода, тегло, чат и AI хранителен режим. Ако трябва синхронизация между телефони или централизирана база за всички потребители, добави Supabase/Firebase backend.

## Какво има вътре

- Профил: пол, възраст, височина, тегло, целево тегло, активност и цел
- Изчисления: BMI, BMR, TDEE, калории, протеини, мазнини, въглехидрати, фибри и вода
- Персонален хранителен план
- Камера/качване на снимка на храна
- Groq Vision анализ със структуриран JSON
- Дневен калориен лимит и оставащи калории
- История на храненията
- Графика за тегло и прогрес
- AI чат за хранителни съвети
- Светла и тъмна тема
- Локално запазване в браузъра

## Стартиране на Windows

Отвори `index.html` директно в браузър.

За камера през браузър е най-добре проектът да се отвори през HTTPS, например през Vercel.

## Качване в GitHub

1. Създай нов repository в GitHub, например `nutriai-web`.
2. Качи всички файлове от тази папка.
3. След това свържи repository-то с Vercel.

Само GitHub Pages не е достатъчен за AI функциите, защото API ключът вече се пази във Vercel serverless функция.

## Groq API ключ

1. Отвори [console.groq.com](https://console.groq.com/).
2. Влез в профила си.
3. Отвори `API Keys`.
4. Създай нов ключ.
5. Постави го във Vercel като environment variable.

Ключът обикновено започва с `gsk_`.

## По-сигурен вариант: GitHub + Vercel

GitHub Pages не може да пази тайно Groq API ключ. Ако искаш ключът да не се въвежда в браузъра:

1. Качи проекта в GitHub.
2. Влез във Vercel и избери `Import Git Repository`.
3. Избери repository-то.
4. В `Environment Variables` добави:
   - `GROQ_API_KEY` = твоят Groq API ключ
5. Deploy.

Файлът `api/groq.js` ще работи като serverless proxy към Groq.

## Vercel режим

Приложението вече не показва поле за API ключ. Ключът се пази във Vercel като `GROQ_API_KEY`, а браузърът използва `/api/groq`.

Това е по-сигурно, защото ключът не се вижда в браузъра.

## Безопасност

Приложението дава ориентировъчни хранителни оценки и не заменя лекар, диетолог или медицински съвет. Не препоръчва опасни крайни диети.
